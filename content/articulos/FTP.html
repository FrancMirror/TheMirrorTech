FTP (File Transfer Protocol) es un protocolo de red para transferir archivos entre sistemas conectados a una red TCP (Transmission Control Protocol), basada en arquitectura cliente-servidor. Desde un ordenador cliente puede conectarse a un servidor, descargar archivos de él o enviar archivos, independientemente del sistema operativo utilizado en cada ordenador.

El protocolo FTP empezó a usarse en abril de 1971, publicado como RFC 114, antes de que existiera el modelo TCP/IP. La estructura general se estableció en 1973. Fue modificado varias veces, agregando nuevos comandos y funcionalidades. Finalmente, en octubre de 1985 se publicó el RFC 959, que es el que se utiliza actualmente.

FTPS & SFTP

¿Qúe es FTPS?

FTPS significa FTP (File Transfer Protocol) sobre SSL/TSL (son protocolos seguros). SSL significa Secure Socket Layer y TSL significa Transfer Layer Security, es el sucesor de SSL. Usa algoritmos más fuertes y es más seguro que SSL. Una tecnología para la comunicación segura de datos entre dos ordenadores a través de Internet. Esta tecnología utiliza certificados digitales y encriptación de datos para verificar la identidad del servidor y garantizar la seguridad de los datos.

¿Qué es SFTP?

El Protocolo seguro de transferencia de archivos o SSH File Transfer Protocol (SFTP) es un versión segura del Protocolo de transferencia de archivos (FTP) y una parte del Protocolo SSH para una fácil transferencia de datos y acceso a datos a través de un flujo de datos Secure Shell (SSH). Proporciona una conexión segura para transferir archivos y atravesar el sistema de archivos tanto en el sistema local como en el remoto. El cifrado en SFTP se realiza mediante una conexión SSH, donde los archivos se pueden transferir usando WinSCP y el cliente SFTP.

Diferencias entre FTPS & SFTP

FTPS SFTP
Basado en FTP Basado en SSH
Puerto 21 or 990 Puerto 22
Necesita certificado SSL No necesita certificado SSL
2 canales: comando y canal datos Usa solo 1 canal
Tiene inicio de sesión anonymous Inicia sesión con una llave
